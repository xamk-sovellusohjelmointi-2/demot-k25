Demo 5: Tietomallin toteutus Prisma ORM:n avulla

Tämä on demo REST API -palvelinsovelluksen toteutuksesta vaihtoehtoisella tavalla. Tähän saakka olemme käyttäneet sovelluksen tietomallina json-tiedostoa ja/tai sen käsittelyyn itse ohjelmoitua luokkaa (class), josta muodostettua oliota käytetään puolestaan reittikäsittelijöissä. Tällaisiin tietomallehin ei kuitenkaan aina tarvitse tehdä itse koodia, vaan voimme käyttää erilaisia valmiita ratkaisuja. Yksi tällainen väline Node-palvelinohjelmoinnissa on Prisma ORM (Object-Relational Mapping). Tässä demossa sovellamme Prismaa ostoslistasovelluksen tietomallina.

Testaa demon toiminnallisuutta Postman-sovelluksella. Luo Postman-sovelluksella erilaisia GET-, POST-, PUT- ja DELETE-pyyntöjä url-osoitteeseen http://localhost:3005/api/ostokset.
